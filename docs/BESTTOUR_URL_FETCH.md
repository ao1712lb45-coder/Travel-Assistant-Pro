# Besttour 行程網址自動抓取

## 功能

在主畫面貼上單一 Besttour 行程網址，例如：

```text
https://www.besttour.com.tw/itinerary/TYO05JX260726PJ
```

按「自動抓取並解析」後，系統會：

1. 透過本機代理讀取 Besttour HTML。
2. 確認回傳內容包含網址中的團號，避免把首頁誤當行程。
3. 解析名稱、團號、天數、價格、日期、航空及亮點。
4. 更新主畫面、三平台文案與 DM。

## 為什麼需要本機代理

瀏覽器會限制網頁直接讀取另一個網域（CORS）。本機代理只接受：

- `https://besttour.com.tw/itinerary/...`
- `https://www.besttour.com.tw/itinerary/...`

並有 15 秒逾時、2 MB 上限、HTML 類型檢查及重新導向後網域檢查，避免任意網址存取。

## 啟動

需要 Node.js 18 以上。Windows 可雙擊：

```text
START_SERVER.bat
```

或執行：

```text
npm start
```

然後開啟 `http://127.0.0.1:4173`。

直接雙擊 `index.html` 時，原有手動貼上官網文字功能仍可使用，但自動抓取不可用。

## 已知限制

- 官網若要求登入、阻擋伺服器請求、或只在瀏覽器執行 JavaScript 後才提供資料，代理可能拿不到完整內容。
- 官網若將行程導回首頁，系統會回報「未回傳這個團號」，不會套用首頁資料。
- 抓取成功不代表每個欄位都有顯示；未確認欄位會提醒人工檢查。
- 此功能只處理使用者主動輸入的一個行程網址，不做大量爬取。
