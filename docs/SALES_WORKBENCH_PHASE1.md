# 旅行社業務工作台：第一階段

## 現有能力盤點

- Besttour／ITTMS 單一團號與網址匯入
- Besttour 官方搜尋、分頁同步、中斷續傳
- 官方每日行程與景點內容搜尋
- 團號、日期、天數、航空、價格、名額及亮點解析
- 本機瀏覽器行程資料庫、JSON 匯入／匯出
- 客戶地區、月份、人數、預算、關鍵字、族群與行動需求配對
- LINE、Facebook、Threads 文案及發布前檢查
- Render 網頁部署與選用的基本密碼保護

## 第一階段新增

- 客戶 LINE 原話結構化：日期、機場、目的地、備選、人數、預算、天數、商品類型與偏好
- 缺失欄位清單及可直接複製的追問 LINE
- 日期範圍、機場、天數、廉航與購物站排除條件
- 無完全符合商品時，依備選目的地、日期、機場、天數、預算及偏好提供替代方案並標示放寬項目
- 2～5 團比較：價格、日期、航班、航空、天數、飯店、購物站、景點、額外費用、成團狀態、來源及更新時間
- 簡短版、三方案比較版、找不到商品／替代方案版 LINE 回覆

## 沿用的架構

- `server.js`：官方 API 代理、安全網域限制及靜態檔案服務
- `src/tour-parser.js`：團號與官網行程解析
- `src/recommendation.js`：官方內容搜尋、本機資料庫與配對排序
- `src/app-shell.js`：單頁業務操作導覽
- `src/sales-workbench.js`：第一階段新增的 LINE 客需、比較與回覆工作台

## 資料限制

- 目前官方資料來源以團體行程為主；自由行與機加酒需外部訂房／機票商品系統。
- 飯店、購物站數量、額外費用及正式成團狀態若官方介面未提供，顯示「待人工確認」。
- 可售名額不等於成團，不能據此推測已成團。
- 坡道、樓梯及無障礙條件只能依官網文字初篩，仍需人工向業務、景點及飯店確認。
- 客戶 LINE 原文只在目前瀏覽器頁面解析，不會寫入 GitHub；搜尋時只送出必要的行程關鍵字。

## 建議後續順序

1. 串接公司正式庫存／訂單／成團狀態系統。
2. 補齊飯店、房型、購物站、領隊小費與自費活動等結構化欄位。
3. 增加業務登入、權限、客戶案件儲存與交接紀錄。
4. 增加人工確認清單與主管覆核流程。
5. 若需要 AI 自由語句理解，再評估付費 API，金鑰只存伺服器環境變數。
